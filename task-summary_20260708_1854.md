# 模拟配送全流程实现

## 目标
实现后端模拟骑手配送全流程，骑手信息固定为"桃之 17622223333"，骑手位置从商家坐标模拟移动到用户坐标，前端实时更新。

## 实现细节

### 1. 后端模拟配送 (`api/delivery_sim.php`)
- **dispatch**: 点击"发配送"后生成派单信息写入订单备注（取货/收货地址、联系人、电话、金额），订单状态变更为 `delivering`，`delivery_status` 设为 `waiting`
- **accept**: 骑手接单 → `delivery_status = 'accepted'`
- **arrived_store**: 骑手到店 → `delivery_status = 'at_store'`，骑手位置更新到商家坐标
- **picked_up**: 骑手取货出发 → `delivery_status = 'delivering'`
- **simulate_route**: 20步插值移动，从商家坐标渐变到用户坐标，每步更新 `rider_lat`/`rider_lng`，间隔1.5秒
- **complete**: 骑手送达 → `status = 'completed'`, `delivery_status = 'delivered'`

### 2. 前端订单详情页 (`pages/order-detail/`)
- **骑手状态卡片**: 显示骑手状态文本（等待接单/已接单/到店/配送中/已送达）
- **骑手信息**: 显示骑手姓名、电话，支持一键拨打
- **地图标记**: 商家、用户、骑手三个marker
- **配送路线**: 配送中显示商家→骑手→用户折线（橙色实线+绿色虚线）
- **自动刷新**: 骑手状态为 `delivering`/`accepted`/`at_store` 时每3秒请求一次后端
- **时间轴**: 根据 `delivery_status` 动态显示配送进度

### 3. 骑手位置模拟算法
```javascript
// 线性插值 + 正弦波动
progress = step / totalSteps;
riderLat = storeLat + (userLat - storeLat) * progress + sin(step * 0.8) * 0.0003;
riderLng = storeLng + (userLng - storeLng) * progress + cos(step * 0.8) * 0.0003;
```
